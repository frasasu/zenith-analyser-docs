Author : François TUMUSAVYEYESU

Abstract

Zenith Analyzer represents an innovative and sophisticated software solution for the analysis, simulation and management of complex temporal structures. This Python library, designed with a modular and extensible architecture, allows developers and data scientists to efficiently manipulate hierarchical temporal laws expressed in the Zenith language specifically created for this domain.

    Important Note : Zenith Analyzer is distributed under the Apache 2.0 license, ensuring free and flexible use while providing adequate legal protection for commercial and open-source projects.

    In the constantly evolving field of temporal data analysis and chronological event modeling, the need for specialized tools capable of handling complex hierarchical structures is becoming increasingly critical. Zenith Analyzer emerges as an elegant and powerful response to this growing need.

    The design philosophy behind Zenith Analyzer revolves around three fundamental principles: modularity, extensibility, and robustness. Each component of the library has been carefully architected to function independently while integrating harmoniously into the overall system.

    Library Architecture

    The architecture of Zenith Analyzer has been meticulously designed according to the principles of separation of concerns and single responsibility. Each module has a well-defined function and clear interfaces with other system components.

    Main modules and their responsibilities

    Module Description and main responsibilities
    analyzers.py Analytical heart of the system, containing the main classes that orchestrate the complete analysis of temporal structures.
    lexer.py Responsible for lexical analysis, transforming the raw character stream into a meaningful sequence of tokens.

    The Zenith Language: Fundamental Concepts

    The Zenith language represents the primary declarative interface between the user and the analysis system. Specifically designed to describe complex temporal structures, it adopts a declarative programming approach where the user specifies what should be modeled rather than how to model it.

    Fundamental structure

    A typical Zenith document consists of a hierarchical collection of laws (law) and targets (target), organized in a tree structure that reflects the logical and temporal relationships of the modeled system.

    Example of a Zenith law

    Zenith 
    law law_name:
    start_date:2024-03-15 at 09:30
    period:2h45
    Event:
        opening_meeting:"Opening meeting and presentation of objectives"
        work_session:"Collaborative work session"
        coffee_break:"Coffee break and networking"
    GROUP:(opening_meeting 45m^10m - work_session 60m^15m - coffee_break 30m^5m)
end_law

    Example of a target with hierarchy

    Zenith 
    target main_project:
    key:"PROJ-2024-001"
    dictionary:
        weekly_review:"Weekly progress and challenges review"
        tech_workshop:"Technical workshop on new technologies"
    
    law phase1_planning:
        start_date:2024-04-01 at 08:00
        period:3h
        Event:
            day_briefing:"Day briefing and task distribution"
            individual_work:"Individual work on assigned tasks"
        GROUP:(day_briefing 30m^0 - individual_work 120m^15m)
    end_law
end_target

    Fundamental temporal concepts

    • Chronocoherence : Effective duration of an event
    • Chronodispersion : Time interval between the end of one event and the start of the next
    • Sequencing : Order of events in a group
    • Temporal completeness : Sum of chronocoherences and chronodispersions

    ZenithAnalyzer: The central entry point

    The ZenithAnalyzer class constitutes the main entry point and central coordinator of the entire analysis infrastructure. It orchestrates the entire processing pipeline.

    Initialization

    Python 
    from zenith.analyzers import ZenithAnalyzer

# Loading source code
with open("model.zen", "r", encoding="utf-8") as source_file:
    zenith_code = source_file.read()

# Creating analyzer instance
analyzer = ZenithAnalyzer(zenith_code)

# Checking for errors
if analyzer.parser_errors:
    print(f"Parsing errors detected: {len(analyzer.parser_errors)}")

    Main methods

    Python 
    # Analysis of a specific law
analysis = analyzer.law_description("sprint_planning")

# Analysis of a target
target_analysis = analyzer.target_description("main_project")

# Analysis by population level
population_analysis = analyzer.population_description(2)

# Complete corpus analysis
corpus_analysis = analyzer.analyze_corpus()

# JSON export
analyzer.export_json("analysis.json")

    The temporal simulation system of Zenith Analyzer relies on a rigorous design that guarantees chronological accuracy while maintaining the flexibility needed to model complex scenarios.

    Duration and interval representation

    Python 
    from zenith.utils import point_to_minutes, minutes_to_point

# Duration conversion
durations = ["30m", "2h", "1h30m", "45", "2h15m"]

for duration_str in durations:
    minutes = point_to_minutes(duration_str)

    Hierarchical Inheritance

    The hierarchical inheritance system of Zenith Analyzer constitutes one of its most powerful and distinctive features. It allows modeling complex relationships between different parts of a temporal system.

    target_analyzer = analyzer.target_analyzer
    
    # Global hierarchical structure
    max_depth = target_analyzer.get_max_generation()

        targets_at_level = target_analyzer.get_targets_by_generation(depth)
        print(f"Level {depth}: {len(targets_at_level)} targets")

    Integration with databases

    Python 
    import sqlite3
import json

class ZenithDatabase:
    """Class for database integration."""
    
    def __init__(self, db_path="zenith_analysis.db"):
        self.db_path = db_path
        self._init_database()
    
    def store_analysis(self, analysis_json):
        """Stores a complete analysis in the database."""
        with sqlite3.connect(self.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute("""
                INSERT INTO analyses (source_file, raw_json) 
                VALUES (?, ?)
            """, (
                analysis_json.get('metadata', {}).get('source_file'),
                json.dumps(analysis_json)
            ))
